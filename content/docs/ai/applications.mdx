This section is where the framework meets the keyboard. It covers **how to phrase a request**, **which tool to pick**, **what realistic use cases look like**, and **how to build a small agent** when a prompt is not enough.

## Searching vs. Prompting

If you were assigned the task of researching *"the history of the Roman Empire"*, where would you start?

The traditional answer is a **search engine** — you type a query, you get a ranked list of web pages, you actively click through them. This is **searching**.

Generative AI introduces a different mode: you **ask a question**, the model retrieves and summarizes the information for you, you have a conversation. This is **prompting**.

| | Searching | Prompting |
|---|---|---|
| **Interaction** | One-shot, ranked list of links | Conversational, iterative |
| **You do** | Click, read, synthesize yourself | Ask, evaluate, refine |
| **Output** | Sources you consult | A synthesized answer (with the same fact-checking burden) |
| **Best for** | Finding the original document | Getting an overview, drafting, summarizing |

Both are useful. Use the right tool for the question.

## Prompt Engineering

> To interact with a generative AI model, we must provide **clear and concise instructions** (prompts). Being able to craft prompts that get the AI to generate the desired results is an extremely useful skill to master. However, there is **no perfect prompt** — it takes practice and experimentation. The process of refining these instructions is called **prompt engineering**.

### Elements of a good prompt

| | Less effective | More effective |
|---|---|---|
| | ✗ Vague | ✓ Specific and detailed |
| | ✗ A few words | ✓ Complete sentences with specific instructions |
| | ✗ No context on the desired result | ✓ Indicates tone, purpose, desired format |
| **Example** | *"Summarize the news about [company name]."* | *"I work in marketing, focusing on competitor research. Give me a concise summary of recent news on [company name]. Focus on announcements of new product lines. Provide the response in two or three paragraphs and use a professional tone."* |
| **Result** | Vaguer than desired, in a format you did not ask for. | What you actually wanted. |

### The Persona–Context–Task–Format framework

A reliable structure for any prompt, with two optional add-ons.

| Component | What it does | Example |
|---|---|---|
| **Persona** | Asks the tool to assume a role and bring its expertise. | *"Act as a project manager."* |
| **Context** | Gives the model the background it needs to understand the situation. | *"I need to inform a key stakeholder about a change in the project timeline for \[project name]. The launch date is moving from \[old date] to \[new date] due to \[reason]."* |
| **Task** | States what you want the AI to do. | *"Draft a clear, concise, and professional email that explains the situation and outlines our plan to mitigate further delays."* |
| **Format** | Tells the model the shape of the output. | *"…as a bulleted list / table / 3-paragraph email."* |
| **References** *(optional)* | Examples or past artifacts the AI can imitate. | *"Attached is further context about the issue / a past stakeholder email."* |

Then, after the model answers, you **evaluate** and **iterate** — refining the prompt until you get what you want.

### A worked example

> *"**Act as a tutor** who breaks down complex topics into simple explanations. I want you to **explain Boolean variables** to **a 22-year-old student, to help them prepare for the Data Science exam**. Your response should be **300 words, written in a friendly and educational tone**."*

Components:

- **Persona** — *Act as a tutor who breaks down complex topics into simple explanations.*
- **Task** — *explain Boolean variables.*
- **Context** — *to a 22-year-old student, to help them prepare for the Data Science exam.*
- **Format** — *300 words, friendly and educational tone.*

### Three practical tips

1. **Give clear instructions** — use action verbs: *explain, translate, summarize, compare*.
2. **Provide context** — name the project type: *short story, report, outline, briefing*.
3. **Iterate and experiment** — prompting is an experiment. Several cycles of iteration are normal and expected.

## Lara — translation

> [laratranslate.com/traduttore](https://laratranslate.com/traduttore)

Lara is the University's approved translation tool. The interface offers three translation modes:

- **Faithful** — closest to the source, preserves structure (legal text, regulation).
- **Fluid** — natural target-language flow (correspondence, internal comms).
- **Creative** — adapts the message (marketing, web copy).

Use Lara instead of free DeepL or browser-extension translators when the source text contains Luiss information.

## Gemini

> [gemini.google.com/app](https://gemini.google.com/app)

Gemini is Google's flagship multimodal model. Inside the chat you can:

- Type or speak a prompt.
- Create images.
- Create videos.
- Use **Tools** for browsing, deep research, code.
- Pick a fast or thinking model from the dropdown.
- Apply **Gems** — your saved agents.

### Use case 1 — comparing three assistants on the same prompt

A "Luiss strengths & weaknesses" prompt run against ChatGPT, M365 Copilot and Gemini will produce three different answers. The most-effective version of the prompt forces the model to **show its work**:

> *"Analyze the strengths and weaknesses of Luiss. Assume the role of a metacognitive reasoning expert. For each complex problem:*
>
> *- **Decompose**: break the problem into sub-problems*
> *- **Solve**: address each one with explicit confidence (0.0–1.0)*
> *- **Verify**: check logic, facts, completeness, and biases*
> *- **Synthesize**: combine using weighted confidence*
> *- **Reflect**: if confidence is < 0.8, identify weak points and retry*
>
> *For simple questions, go directly to the answer. Always respond by providing: a clear answer, the confidence level, key caveats. You must provide a summary of the problems listed above. The problem is as follows: Analyze the strengths and weaknesses of Luiss."*

This pattern (Decompose / Solve / Verify / Synthesize / Reflect) is reusable for any analytical task.

### Use case 2 — drafting email responses with a reusable template

A reusable prompt for an executive assistant. Run twice — first to declare the **schema**, then to fill it in.

```text
Step 1 — declare the structure

I want you to act as an executive assistant expert in communication management.
The request is to draft responses for the following emails:

Email content     — [Summary or key points of the received email]
Response objective — [Goal of the response, e.g. Information, Confirmation, Rejection]
Tone              — [Desired tone, e.g. Formal, Informal, Professional]
Urgency level     — [Urgency level, e.g. Low, Medium, High, Urgent ASAP]
Language          — [Italian, English, Chinese, etc.]

Formatting guidelines:
"Write a concise and clear email response that addresses all the key points
raised, is consistent with the desired tone, and meets the response objectives."

Step 2 — fill it in

Email content — [paste the email]
Response objective — Politely decline and ask whether a 3-month POC under €4,999
                     is feasible
Tone — Professional
Urgency level — Medium
```

The model returns a clean draft that you then review, edit and send.

### Use case 3 — deep research

Gemini's deep-research mode (or the equivalent in Copilot) can produce structured analyses. Example prompt for analyzing AI in higher education:

```text
Task:       Systematically analyze the use of AI in Teaching & Learning at
            leading universities at national and international level, including
            AI projects developed with a Digital Transformation approach.

Persona:    Act as an expert in digital transformation in the university sector
            at Luiss Guido Carli, with advanced skills in AI applied to
            education, instructional design, academic policy and organizational
            innovation.

Context:    The analysis aims to understand how universities are integrating AI
            into teaching models, student services and institutional processes,
            to support Luiss Guido Carli's strategic decisions in academic
            development and digital innovation.

Format:     Structure the output in clear, comparable sections, including:
            - Overview of emerging trends
            - Case studies from relevant universities
            - Types of AI applications in Teaching & Learning
            - Impacts on students, faculty and governance
            - Challenges, risks and ethical implications
            - Best practices and replicable models
            Use analytical and professional language, with lists and tables.
```

## NotebookLM — turning your documents into anything

> [notebooklm.google.com](https://notebooklm.google.com/)

NotebookLM is structured around **your own sources**. You upload PDFs, websites, text or Drive files; the tool can chat about them and produce a variety of outputs.

### Why NotebookLM is different — from LLM to RAG

A plain LLM answers from its pretrained knowledge. The downside:

- May lack **domain knowledge** specific to your organization.
- May not be **up to date** — the training cutoff has passed.
- **Cannot cite** original sources.

**Retrieval-Augmented Generation (RAG)** solves this. RAG lets an LLM retrieve information from an **external knowledge base** (your documents, data, media) before generating a response. The retrieved snippets are provided to the LLM as context. This **increases accuracy** and **reduces hallucinations**.

The flow:

```
1. User asks the LLM a question.                  →  "How do customers perceive
2. Behind the scenes, a Retriever finds the         our services recently?"
   most relevant chunks in your documents.
3. LLM generates an answer using the retrieved
   chunks as grounded context.
4. User receives the answer (with citations).
```

### RAG components

- **Embeddings** — floating-point vectors that represent text or other data. Embeddings capture **semantic meaning and context**; text with similar meaning has closer embeddings.
- **Vector search** — store data in a specialized vector database, optimized for fast similarity lookups.
- **LLM** — send retrieved document chunks to a model (e.g. Gemini) to summarize a response.

A multimodal embedding model can embed text, images and videos into the **same** vector space, so a text query can retrieve a relevant image and vice versa.

### What you can generate in NotebookLM

From a set of uploaded sources, NotebookLM's Studio panel produces:

- **Audio Overview** — a two-host podcast discussing the material.
- **Video Overview** — a narrated video summary with slides.
- **Mind Map** — an expandable knowledge tree of the concepts.
- **Reports** — Briefing Doc, Study Guide, Blog Post, Policy Briefing, Strategic Plan, Explanatory Guide, Concept Explainer.
- **Flashcards** — Q/A pairs for spaced-repetition study.
- **Quiz** — multiple-choice questions for self-assessment.
- **Infographic** *(beta)* — a single-page visual summary.
- **Slide Deck** *(beta)* — a presentation you can download.

### Use case — analyzing university policies

A typical NotebookLM notebook for institutional work might collect:

- *Allegato n. 1 — Sistema Disciplinare*
- *Atto organizzativo Luiss in materia Whistleblowing*
- *Codice di comportamento degli studenti Luiss*
- *Codice di deontologia — buona condotta scientifica*
- *Codice Etico Luiss*
- *Informazioni da inserire nei documenti da mandare*
- *Linee Guida per un Linguaggio Inclusivo*
- *Luiss — Procedura di ricezione e gestione reclami*

You can then ask Studio to produce, for instance, an **Audio Overview** titled *"Simplifying the regulatory maze"*, or generate **Flashcards** of the disciplinary system, or have a **Quiz** based on the AI Act for staff onboarding.

## From prompts to agents

> **Agents : AI = Apps : Smartphones**

A **prompt** is a single instruction. An **agent** is a *configured assistant* — a persona + instructions + reference knowledge + (optionally) tools — that you can call by name to do a recurring job. The same way an iPhone is just a screen until you install apps, the LLM is just a chatbot until you turn it into agents that automate your repetitive work.

### Building a Gemini Gem

A **Gem** is Google Gemini's agent unit.

1. Open the Gemini sidebar → **Gems** → **+ New Gem**.
2. **Name** — short and descriptive (e.g. *GEO Optimizer*).
3. **Description** — one-liner shown to the user.
4. **Instructions** — the system prompt that runs every time the Gem is invoked. This is the most important field; it defines persona, task, output rules.
5. **Knowledge** — reference sources (web URLs, PDFs, Word documents). The Gem consults these to ground its answers.
6. **Update** — save.

Google's official guidance on writing agent prompts: [docs.cloud.google.com — agent designer best practices](https://docs.cloud.google.com/gemini/enterprise/docs/agent-designer/create-agent?hl=it#best-practice-prompt).

### Building a Copilot Studio agent

In M365 Copilot → **Create agent** → **Copilot Studio**.

1. **Describe / Configure / Try it** — three tabs in the builder.
2. **Name** — agent identifier.
3. **Description** — what the agent does and for whom.
4. **Instructions** — system prompt. Describe what the agent should do, define its tone, outline its rules.
5. **Knowledge** — URLs, files, SharePoint, Teams, Outlook emails. You can prioritize the agent's own knowledge over generic Copilot answers.
6. **Capabilities** — toggles for Code interpreter, Image generator.
7. **Suggested prompts** — 3–4 example prompts visible to the user when they open the agent (e.g. *"What are the company policies regarding device assignment?"*).

### Use case — turning the email assistant into a permanent agent

The email-response prompt from earlier can be promoted to a Gem or Copilot Studio agent:

| Field | Content |
|---|---|
| **Name** | Email from Notes |
| **Instructions** | The "executive assistant" prompt from [Use case 2](#use-case-2--drafting-email-responses-with-a-reusable-template) above, with placeholders for email content, objective, tone, urgency, language. |
| **Knowledge** | Past stakeholder emails (optional), tone-of-voice guide. |
| **Suggested prompts** | *"Decline politely and propose a smaller POC", "Confirm receipt and ask for next steps", "Forward to the right team"* |

After this is built, you no longer paste the long system prompt every time — you just open the agent, drop the email body in, and it returns the draft.

## Key takeaways

1. **Searching** finds documents. **Prompting** synthesizes answers. Both still need fact-checking.
2. The **Persona–Context–Task–Format** prompt structure is the difference between vague output and usable output.
3. Pick the tool that fits the task: **Lara** for translations, **Gemini / Copilot** for productivity and research, **NotebookLM** for working with *your own* documents, **Copilot Studio / Gems** for automation.
4. When a prompt becomes a recurring workflow, turn it into an **agent**.
